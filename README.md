# home-ssl-lab

本仓库用于归档和发布多个独立 HTML 页面。这些页面不针对任何单一域名或具体网站，而是作为通用测试、演示或说明用途的静态资源集合。

## 项目简介

`home-ssl-lab` 是一个轻量级的静态页面仓库，主要用于存放和展示与 SSL/TLS 相关（或无关）的独立 HTML 文件。每个页面均可单独访问，适合用于本地实验、功能验证或作为示例参考。

仓库本身不依赖任何后端服务，所有内容均为纯前端静态页面。

## 目录说明

```
home-ssl-lab/
├── README.md          # 本文件
├── index.html         # 仓库首页（可选）
├── pages/             # 存放独立 HTML 页面的目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/            # 静态资源（如图片、样式、脚本等）
    ├── css/
    ├── js/
    └── images/
```

- `pages/`：主要归档目录，每个 `.html` 文件为一个独立页面。
- `assets/`：存放页面所需的公共资源，按类型分文件夹管理。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开。
- 页面之间可能存在链接或引用关系，但均在同一仓库内完成。
- 每个页面头部通常包含简要说明或版本标注，以便快速了解用途。
- 不包含任何外部跟踪、广告或第三方内容。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并在必要时更新 `index.html` 中的导航链接。
- 建议为每个页面添加清晰的标题和注释，方便后续维护。
- 公共资源（如 CSS、JS）尽量复用 `assets/` 中的文件，减少重复。
- 仓库采用常规 Git 进行版本管理，提交信息请简要描述变更内容。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/home-ssl-lab.git
   ```
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件，或使用本地 HTTP 服务器预览（例如 `python -m http.server`）。
